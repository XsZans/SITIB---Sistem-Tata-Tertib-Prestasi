BAB 3
ROLE & HAK AKSES

3.1. Penjelasan Role Pengguna

Tabel role dan hak akses:

Role                Hak Akses Utama
Admin               Full akses semua fitur, kelola user, kelola data master, backup database, verifikasi user
Guru BK             Bimbingan konseling (BK Session), approve/reject pengajuan bimbingan, export laporan BK
Wali Kelas          Input pelanggaran siswa di kelas, lihat data siswa kelas, cetak laporan kelas
Guru                Input pelanggaran (perlu verifikasi), request laporan, lihat data siswa
Kesiswaan           Verifikasi pelanggaran & prestasi, kelola sanksi, monitoring keseluruhan
Kepala Sekolah      Monitoring & laporan keseluruhan sekolah, dashboard statistik, view only
Siswa               Lihat pelanggaran & prestasi sendiri, ajukan bimbingan BK, respon undangan BK
Orang Tua           Monitoring pelanggaran & prestasi anak, terima notifikasi, cetak laporan anak

3.2. Penjelasan Detail Setiap Role

3.2.1. Admin
- Kelola data siswa (tambah, edit, hapus, lihat)
- Kelola data guru dan wali kelas (tambah, edit, hapus, lihat)
- Kelola data orang tua (tambah, edit, hapus, lihat)
- Kelola jenis pelanggaran dan poin (tambah, edit, hapus, lihat)
- Kelola jenis prestasi dan poin (tambah, edit, hapus, lihat)
- Kelola jenis sanksi (tambah, edit, hapus, lihat)
- Input dan kelola pelanggaran siswa
- Input dan kelola prestasi siswa
- Verifikasi user baru (approve/reject)
- Verifikasi laporan pelanggaran dan prestasi (approve/reject dengan alasan)
- Verifikasi request laporan dari guru (approve/reject)
- Kelola pelaksanaan sanksi
- Proses sanksi berdasarkan poin pelanggaran
- Selesaikan sanksi dengan upload bukti
- Input panggilan BK untuk siswa
- Buat akun user untuk guru, siswa, dan orang tua
- Backup database (otomatis dan manual)
- Restore database dari backup
- Setting backup otomatis (harian/mingguan/bulanan)
- Lihat riwayat backup
- Delete backup dengan verifikasi password
- Lihat semua laporan dan statistik
- Export laporan pelanggaran dan prestasi ke PDF
- Filter dan search data siswa, guru
- Load more data siswa dan guru (pagination)
- Grafik pelanggaran dan prestasi per bulan
- Lihat detail pelanggaran per siswa
- Lihat data tidak terverifikasi (ditolak)
- Akses penuh ke semua menu

3.2.2. Guru BK
- Buat sesi bimbingan konseling (BK Session)
- Undang siswa untuk mengikuti bimbingan
- Konfirmasi jadwal bimbingan
- Input hasil dan catatan bimbingan
- Selesaikan sesi bimbingan (complete session)
- Batalkan sesi bimbingan (reject session)
- Approve pengajuan bimbingan dari siswa
- Reject pengajuan bimbingan dari siswa
- Lihat notifikasi BK
- Mark notifikasi sebagai sudah dibaca
- Lihat riwayat bimbingan (selesai/dibatalkan)
- Filter riwayat berdasarkan status dan search siswa
- Search siswa untuk bimbingan
- Filter siswa berdasarkan kelas dan jurusan
- Export laporan bimbingan konseling ke PDF
- Lihat pending sessions (sesi yang menunggu)
- Lihat today sessions (sesi hari ini)
- Lihat completed sessions (sesi selesai)

3.2.3. Wali Kelas
- Input pelanggaran siswa di kelas yang diampu (status: menunggu verifikasi)
- Lihat data lengkap siswa di kelas
- Lihat riwayat pelanggaran siswa di kelas
- Lihat riwayat prestasi siswa di kelas
- Lihat poin pelanggaran dan prestasi siswa kelas
- Monitoring kedisiplinan siswa di kelas
- Get jenis pelanggaran berdasarkan kategori
- Cetak laporan kelas (per siswa atau keseluruhan)
- Export laporan kelas ke PDF
- Simpan laporan untuk arsip (auto save ke storage)
- Lihat laporan tersimpan
- Download laporan yang tersimpan
- Filter siswa berdasarkan nama, NIS, dan sort

3.2.4. Guru
- Lihat data siswa (semua siswa)
- Search dan sort data siswa
- Input pelanggaran siswa (status: menunggu verifikasi)
- Get jenis pelanggaran berdasarkan kategori
- Request laporan siswa (perlu verifikasi admin)
- Lihat status request laporan (pending/approved/rejected)
- Download laporan yang sudah diapprove
- Lihat laporan tersimpan (storage)
- Auto download report dari storage
- Export laporan siswa ke PDF
- Lihat total siswa dan total pelanggaran
- Lihat new approved reports (laporan baru yang disetujui)

3.2.5. Kesiswaan
- Lihat data siswa (dengan filter dan search)
- Load more siswa (pagination)
- Input pelanggaran siswa (status: menunggu verifikasi)
- Input prestasi siswa (status: menunggu verifikasi)
- Verifikasi pelanggaran yang diinput (approve/reject dengan alasan)
- Verifikasi prestasi yang diinput (approve/reject dengan alasan)
- Lihat semua pelanggaran siswa
- Lihat semua prestasi siswa
- Lihat jenis pelanggaran dan poin
- Lihat jenis prestasi dan poin
- Kelola sanksi siswa
- Proses sanksi berdasarkan poin
- Lihat pelaksanaan sanksi
- Selesaikan sanksi dengan upload bukti
- Hapus pelanggaran siswa
- Lihat statistik pelanggaran per kelas
- Lihat statistik pelanggaran per jenis
- Monitoring siswa dengan poin tinggi
- Lihat laporan keseluruhan sekolah
- Lihat data guru
- Lihat data orang tua
- Kelola data orang tua
- Export laporan pelanggaran dan prestasi ke PDF
- Grafik pelanggaran dan prestasi per bulan
- Lihat data tidak terverifikasi (ditolak)
- Filter siswa prestasi
- Verifikasi request laporan dari guru

3.2.6. Kepala Sekolah
- Lihat dashboard monitoring keseluruhan sekolah
- Lihat data siswa (dengan filter dan search)
- Load more siswa (pagination)
- Lihat detail pelanggaran per siswa
- Lihat semua laporan pelanggaran dan prestasi
- Lihat grafik pelanggaran per bulan
- Lihat grafik prestasi per bulan
- Lihat statistik per kelas
- Lihat statistik per jenis pelanggaran
- Lihat top 10 siswa bermasalah
- Lihat top 10 siswa berprestasi
- Export laporan untuk rapat
- View only (tidak bisa input/edit/delete)

3.2.7. Siswa
- Lihat dashboard dengan statistik pribadi
- Lihat pelanggaran diri sendiri (kecuali yang ditolak)
- Lihat prestasi diri sendiri (kecuali yang ditolak)
- Lihat poin pelanggaran dan prestasi
- Lihat sanksi yang diterima
- Lihat status pelaksanaan sanksi
- Lihat sesi bimbingan BK
- Ajukan bimbingan BK ke guru BK (pengajuan siswa)
- Respon undangan bimbingan BK (diterima/ditolak dengan alasan)
- Lihat jadwal bimbingan
- Lihat catatan hasil bimbingan
- Lihat notifikasi BK
- Mark notifikasi BK sebagai sudah dibaca
- Lihat profil lengkap dengan data orang tua
- Export laporan pribadi ke PDF (pelanggaran, prestasi, sanksi)
- Export laporan BK ke PDF

3.2.8. Orang Tua
- Lihat dashboard dengan statistik anak
- Lihat pelanggaran anak (kecuali yang ditolak)
- Filter pelanggaran berdasarkan status
- Lihat prestasi anak (kecuali yang ditolak)
- Filter prestasi berdasarkan status
- Lihat poin pelanggaran dan prestasi anak
- Lihat pelanggaran selesai (yang sudah diverifikasi)
- Lihat prestasi selesai (yang sudah diverifikasi)
- Lihat sanksi yang diterima anak
- Lihat jadwal bimbingan anak
- Lihat catatan hasil bimbingan anak
- Export laporan perkembangan anak ke PDF (pelanggaran, prestasi, sanksi)

3.3. Alur Verifikasi Pengguna

Proses registrasi hingga akun aktif:

1. User melakukan registrasi di halaman register
2. Sistem mengirim email verifikasi
3. User klik link verifikasi di email
4. Admin melakukan verifikasi akun (approve/reject)
5. User menerima notifikasi hasil verifikasi
6. Jika approved, user dapat login ke aplikasi

Flowchart: Registrasi → Verifikasi Email → Verifikasi Admin → Akun Aktif
