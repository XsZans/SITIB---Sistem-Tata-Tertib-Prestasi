BAB 3
ROLE & HAK AKSES

3.1. Penjelasan Role Pengguna

Tabel role dan hak akses:

Role                Hak Akses Utama
Admin               Full akses semua fitur, kelola user, kelola data master, backup database, verifikasi user
Guru BK             Bimbingan konseling (BK Session), approve/reject pengajuan bimbingan, kirim notifikasi, lihat riwayat
Wali Kelas          Input pelanggaran siswa di kelas, lihat data siswa kelas, cetak laporan kelas
Guru                Lihat data siswa, input pelanggaran (terbatas), lihat laporan
Kesiswaan           Verifikasi pelanggaran & prestasi, monitoring keseluruhan, laporan statistik
Kepala Sekolah      Monitoring & laporan keseluruhan sekolah, dashboard statistik, view only
Siswa               Lihat pelanggaran & prestasi sendiri, respon undangan BK, lihat sanksi
Orang Tua           Monitoring pelanggaran & prestasi anak, terima notifikasi, cetak laporan anak

3.2. Penjelasan Detail Setiap Role

3.2.1. Admin
- Kelola data master (siswa, guru, orang tua)
- Kelola jenis pelanggaran, prestasi, dan sanksi
- Input pelanggaran dan prestasi
- Verifikasi user baru
- Verifikasi laporan pelanggaran dan prestasi
- Kelola pelaksanaan sanksi
- Buat akun user untuk guru, siswa, dan orang tua
- Backup dan restore database
- Export laporan ke PDF
- Lihat grafik dan statistik

3.2.2. Guru BK
- Buat sesi bimbingan konseling
- Undang siswa untuk bimbingan
- Input hasil bimbingan
- Approve/reject pengajuan bimbingan dari siswa
- Lihat riwayat bimbingan
- Export laporan BK ke PDF

3.2.3. Wali Kelas
- Input pelanggaran siswa di kelas
- Lihat data siswa di kelas
- Lihat riwayat pelanggaran dan prestasi kelas
- Monitoring kedisiplinan kelas
- Export laporan kelas ke PDF
- Simpan dan download laporan

3.2.4. Guru
- Lihat data siswa
- Input pelanggaran (perlu verifikasi)
- Request laporan siswa (perlu approve admin)
- Download laporan yang sudah diapprove

3.2.5. Kesiswaan
- Verifikasi pelanggaran dan prestasi (approve/reject)
- Kelola sanksi siswa
- Monitoring siswa dengan poin tinggi
- Lihat statistik dan laporan keseluruhan
- Export laporan ke PDF

3.2.6. Kepala Sekolah
- Lihat dashboard monitoring keseluruhan
- Lihat laporan dan statistik sekolah
- Lihat grafik pelanggaran dan prestasi
- View only (tidak bisa input/edit)

3.2.7. Siswa
- Lihat pelanggaran dan prestasi sendiri
- Lihat poin dan sanksi
- Ajukan bimbingan BK
- Respon undangan BK (hadir/tidak hadir)
- Lihat hasil bimbingan
- Export laporan pribadi

3.2.8. Orang Tua
- Lihat pelanggaran dan prestasi anak
- Lihat poin dan sanksi anak
- Terima notifikasi via email
- Lihat jadwal dan hasil bimbingan anak
- Export laporan anak

3.3. Alur Verifikasi Pengguna

Proses registrasi hingga akun aktif:

1. User melakukan registrasi di halaman register
2. Akun berhasil dibuat dengan status belum terverifikasi
3. Admin melakukan verifikasi akun di menu "Verifikasi User" (approve/reject)
4. Jika approved, user dapat login ke aplikasi
5. Jika rejected, akun akan dihapus

Catatan:
- Jika akun dibuat oleh Admin, langsung terverifikasi otomatis
- User yang belum diverifikasi tidak bisa login

Flowchart: Registrasi → Verifikasi Admin → Akun Aktif
